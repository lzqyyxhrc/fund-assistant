一、项目概述

本项目是一个基于 Streamlit 构建的基金投资管理系统，支持基金持仓管理、自动定投、再平衡策略、指数对比分析、净值走势跟踪等功能。系统采用 SQLite 作为结构化数据存储，使用 AkShare 和 天天基金网 作为数据源。

主要功能模块

📊
持仓管理与资产概览
💰
自动定投与待确认份额管理
📈
指数对比与策略回测
📋
基金净值历史与走势分析
🔒
隐私模式（隐藏敏感金额）
📝
投资日报（AI 生成 + 飞书推送）

五、核心模块详解

5.1 应用入口 app.py

职责：Streamlit 页面初始化、净值加载、主面板渲染

页面刷新 ↓ 1. 调用 load_config() 从 SQLite 读取持仓配置 2. 收集所有基金代码 → get_fund_batch_net_value() 拉取净值 ├─ AkShare 优先 → 天天基金兜底 → 历史缓存兜底 3. sync_today_pending_orders_from_history() 同步今日定投到待确认 4. confirm_pending_shares() 检查净值日期匹配的订单，自动确认份额 5. calculate_category_value() 计算各类市值 6. render_dashboard() 渲染主面板
核心数据流向： SQLite (db) ├→ storage.load_config() → config (持仓+目标) ├→ net_value_history (历史净值) └→ invest_records/transactions (定投历史) AkShare / 天天基金网 └→ fund_fetcher.get_fund_batch_net_value() → net_values (当日净值) components.render_dashboard() ├→ 资产概览（饼图/柱状图） ├→ 持仓明细表格（支持编辑模式） ├→ 定投计划（支持编辑模式） ├→ 定投历史（近5日） ├→ 基金净值走势 └→ 指数对比与策略回测

5.2 基金净值获取 services/fund_fetcher.py

双数据源+缓存机制：
  1. 检查本地缓存（net_value_history）是否为今日数据 → 命中直接返回
  2. 尝试 AkShare (fund_em_open_fund_info_em)
  3. 失败 → 尝试天天基金网 HTTP API
  4. 都失败 → 返回历史缓存兜底
缓存策略：A股基金净值日期=今日即命中；美股/QDII 净值日期=昨日也可命中（美股交易时间滞后）

5.3 定投执行 services/auto_invest.py

函数调用时机作用
execute_auto_invest()早上10点（或手动）按 auto_invest_plans 配置，为每只基金创建 pending_orders，记录到 invest_transactions
confirm_pending_shares()晚上10点（或页面刷新时）遍历 pending_orders，若 net_value_date 的净值已可用，则 实际份额 = 金额 / 净值日期净值，累加到 shares，同时摊薄 cost_price
✔ 幂等性保障：
  • has_invested_today() 检查当日是否已有 invest_records
  • auto_invest_config.last_invest_date 冗余检查
📅 T+2 确认机制：
  • T日 10:00 定投 → 创建 pending_orders (pending_date=T, net_value_date=T)
  • T日 22:00 净值公布 → confirm_pending_shares() 确认份额（A股基金通常T日晚可确认）
  • QDII 基金可能需 T+1 或 T+2 才能拿到净值 → 订单保持 pending 直到可用

5.4 再平衡计算 services/calculator.py

calculate_rebalancing_amounts(targets, category_values, total_value, new_investment, threshold) ├─ 计算 current_weights = 各类别市值 / 总市值 ├─ 计算 deviation = current_weight - target_weight ├─ 若 max(abs(deviation)) > threshold → 需要再平衡 └─ 再平衡时： - 找出 underweight（仓位不足）类别 - 新投入资金优先分配给偏离最大的类别 - 分配权重 = (缺口金额 / 总缺口) × (1 + |偏离度| × 2) - 归一化确保总分配 = new_investment

5.5 仪表盘主面板 components/dashboard.py

页面布局（自上而下）：

  1. 🔒 隐私模式开关 - 开启后金额显示为 ****
  2. 📈 资产概览 - 总资产/持仓成本/持仓收益/基金数量四宫格，各类资产收益卡片，昨日收益分解
  3. 饼图 + 柱状图 - 当前资产分布 + 当前 vs 目标仓位
  4. 持仓明细（支持编辑模式）
    • 编辑模式开关 → 激活后可直接编辑 name/code/shares/cost_price
    • 每只基金一行，支持"删除"按钮 / "+ 添加基金"按钮
    • "保存修改"按钮（编辑完成后一次性写入）
  5. 📈 基金净值走势 - 基金选择 + 时间范围（今年以来/成立以来），Plotly 折线图含 MA120/MA250 均线
  6. 📊 指数对比分析 - 纳指100/红利低波/黄金三条指数曲线，对比一次性买入 vs 定投组合（无再平衡）vs 定投组合（带再平衡）
  7. 定投计划（支持编辑模式） - 每只基金每日定投金额，总额汇总
  8. 定投历史（近5日） - 每条交易明细，⏳待确认 / ✅已确认状态

5.6 指数策略回测 components/index_chart.py

回测逻辑（backtest_dca_with_rebalancing）：

  • 输入：三大指数历史收盘价（从 index_prices 表读取）、目标权重（40%/40%/20%）、再平衡阈值5%、月度定投金额
  • 算法：指数标准化→模拟按月定投→每日检查仓位偏离→超阈值再平衡→计算总资产曲线/夏普比率/最大回撤
  • 输出：一次性买入组合（基准）、定投组合（无再平衡）、定投组合（5%偏差再平衡）
  • 指数数据来源：纳斯达克100(NDX)、红利低波ETF(512890，前复权替代全收益)、黄金价格(AU9999)

5.7 定时任务 auto_invest_cron.py

⏰ 建议的操作系统定时任务配置：
  • 每日 10:00（周一到周五）：python auto_invest_cron.py --invest
  • 每日 22:00（周一到周五）：python auto_invest_cron.py --confirm --report -k YOUR_DEEPSEEK_KEY -f FEISHU_WEBHOOK
命令行参数：
  • --invest / -i：执行定投
  • --confirm / -c：确认到期的待确认份额
  • --report / -r：生成投资日报
  • --api-key / -k：DeepSeek API Key
  • --feishu / -f：飞书 Webhook URL

5.8 日报生成 daily_report.py

模式触发条件内容
简单报告无 API Key纯本地计算，表格化展示市值/仓位/偏离度/各基金盈亏
AI 报告提供 DeepSeek API Key调用 deepseek-v4-pro 模型生成包含市场分析、投资建议、风险提示的专业报告

支持飞书机器人 Webhook 推送。

6.1 日常定投流程

6.2 页面刷新时的数据加载

用户访问 / 刷新页面 ↓ 1. load_config() ← SQLite (fund_positions, pending_orders, targets) 2. 收集所有基金代码 3. get_fund_batch_net_value(all_codes) ├─ AkShare → 缓存 → SQLite fund_nav_history └─ 失败则用历史缓存兜底 4. sync_today_pending_orders_from_history() ← 从 invest_transactions 同步今日定投到 pending_orders 5. confirm_pending_shares() ← 有净值日期匹配的订单自动确认 6. 渲染 dashboard ├→ 计算各类别市值、昨日收益、偏离度 ├→ 生成饼图/柱状图 ├→ 读取 invest_records 展示近5日定投历史 └→ 读取 index_prices 生成指数对比图

九、隐私模式

components/dashboard.py 顶部的 🔒 隐私模式 开关：

  • 开启后 st.session_state["privacy_mode"] = True
  • format_money(amount) 函数将金额格式化为 **** 而非 ¥123,456.78
  • 影响范围：总资产、持仓成本、持仓市值、各基金盈亏、定投金额、昨日收益等所有涉及金额的字段
  • 不影响：基金名称、代码、份额、净值、收益率百分比（仍显示）

十、技术债与改进方向

  1. 时区与净值日期判断：当前根据基金代码在 nasdaq 类别判断为美股，未区分跨市场基金的净值公布时间差
  2. 再平衡仅通过新增资金实现：当前不卖出超配类别，仅通过新增投入向低配类别倾斜，属于"温和再平衡"
  3. 定投历史的状态判断：通过检查 pending_orders 是否仍存在来判断 ⏳/✅，但 invest_transactions.status 字段未被更新同步
  4. 指数数据更新：index_prices 数据目前依赖 CSV 文件手动导入，暂无每日自动更新机制
  5. 无用户认证：单用户本地部署，无多用户/登录体系
  6. 回测不考虑实际基金：使用指数价格模拟，与实际基金（含费率、跟踪误差）的表现有偏差
